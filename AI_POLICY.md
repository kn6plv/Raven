AI Policy
===

1. If you make an AI code or issue submission, this must be clearly marked with the specific AI model used to generate or assist.
2. Any submission must be owned by you. You must understand what the code does, explain it clearly, and be willing
to maintain the code in the future.
3. You need to be the first person reading and understanding what the code does, not me. AI makes mistakes and I want to know you
think it's good to go, not what the AI says.
4. AIs are notoriously verbose and tiresome when submitting to GitHub. A human must write the submission summary.
5. This is a personal project and the author reserves the right to include or reject code as they see fit.
